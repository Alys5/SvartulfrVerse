## Extracted Concepts

### Communication Format Taxonomy

| Type | Format | Use Case |
|------|--------|----------|
| Digital Messages | `[HH:MM AM/PM] **Name** [Emoji]: \`Message\`` | Texts, chats |
| Forums | Thread indents with vote counts | Network discussions |
| Social Media | Tags and hashtags | Posts and comments |
| Email | Header fields + formal body | Corporate communications |
| Notes | All-caps or italics for emphasis | Physical paper |
| Terminal | Backticks for screen text | Sci-fi interfaces |

### Bridge Rule

> Always introduce a comms block with a narrative prose sentence. Never stack blocks back-to-back.

This is a writing methodology rule, not canon.

### Implicit System Commands

Instead of OOC breaking immersion:
```
[System Note: The following scene takes place entirely via network direct messages.]
```

This is a runtime technique, not canonical data.
