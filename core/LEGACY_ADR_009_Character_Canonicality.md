# ADR 009: Character Canonicality

## Context

Experiences frequently recast existing Characters into different narrative roles.

Without clear rules this creates duplicate identities and fragmented canon.

## Decision

A Character is singular.

There is only one canonical version of each Character.

## Rules

There is only one:

- C_Alyssa
- C_Erik
- C_Jasper
- C_Logan
- C_Malachia
- C_Noah
- C_Wulfnic

Experiences never create alternate copies.

Experiences only change casting.

## World Adaptation

A Character may possess different world-specific descriptions.

Example:

Wulfnic

Contemporary:
Patriarch

Urban Fantasy:
Alpha of Alphas

Norse Mythic:
Sovereign

These are world expressions of the same canonical entity.

## POV Override Compatibility

A Character assigned to the POV Slot remains the same Character.

Only their narrative role changes.

## Consequences

Canonical identity remains stable.

Worlds remain reusable.

Experiences remain lightweight.

The repository avoids identity duplication.

## Principle

Characters are canonical.

Worlds provide context.

Experiences provide casting.
